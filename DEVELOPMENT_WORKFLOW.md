# Рабочий процесс разработки

## 🎯 Цель
Безопасная разработка проекта с сохранением данных в продакшене на Hostinger, при этом возможность тестирования локально.

## 📋 Ежедневный рабочий процесс

### 1. Локальная разработка
```bash
# Запуск в тестовом режиме
npm run dev              # React dev server (порт 5173)
cd server && npm run dev # Express dev server (порт 3001)
```

### 2. Тестирование изменений
```bash
# Проверка структуры БД
node database_check.mjs

# Проверка совместимости
node check_compatibility.mjs

# Тестирование новых функций
npm run build
cd server && npm start
```

### 3. Подготовка к деплою
```bash
# Полная очистка локальной БД (если нужно)
node clean_database.mjs

# Подготовка к деплою
node prepare_deployment.mjs

# Безопасный деплой
node safe_deploy.mjs
```

## 🛡️ Защита данных

### База данных
- **Локальная БД**: `enoterra_erp.db` (тестовая, можно очищать)
- **Серверная БД**: `server/enoterra_erp.db` (локальная копия)
- **Продакшен БД**: `public_html/server/enoterra_erp.db` (защищенная)

### Правила работы с БД
1. **Никогда не загружайте** локальную БД на продакшен
2. **Всегда делайте резервную копию** перед деплоем
3. **Тестируйте миграции** на копии продакшен БД
4. **Проверяйте совместимость** перед деплоем

## 🔄 Процесс деплоя

### Шаг 1: Локальная разработка
1. Разрабатываете новые функции локально
2. Тестируете на локальной БД
3. Проверяете совместимость

### Шаг 2: Подготовка к деплою
```bash
# Создание резервных копий
node safe_deploy.mjs

# Проверка совместимости
node check_compatibility.mjs
```

### Шаг 3: Деплой на Hostinger
1. Загружаете файлы на сервер (кроме БД)
2. Устанавливаете зависимости
3. Запускаете миграции (если нужно)
4. Перезапускаете сервер

### Шаг 4: Проверка
1. Проверяете работу приложения
2. Проверяете сохранность данных
3. Мониторите логи

## 📁 Структура файлов

### Локальная разработка
```
enoterra_erp/
├── src/                    # React приложение
├── server/                 # Express сервер
│   ├── index.js           # Основной сервер
│   ├── enoterra_erp.db   # Локальная БД
│   └── uploads/           # Загрузки
├── database_*.mjs         # Утилиты БД
├── clean_database.mjs     # Очистка БД
├── check_compatibility.mjs # Проверка совместимости
├── safe_deploy.mjs        # Безопасный деплой
├── prepare_deployment.mjs  # Подготовка к деплою
└── package.json           # Зависимости
```

### Продакшен на Hostinger
```
public_html/
├── dist/                   # Собранное React приложение
├── server/                 # Express сервер
│   ├── index.js           # Основной сервер
│   ├── enoterra_erp.db   # Продакшен БД (с данными)
│   └── uploads/           # Загрузки
└── .env                   # Продакшен настройки
```

## 🔧 Конфигурация

### Локальная разработка (.env)
```env
NODE_ENV=development
PORT=3001
DB_PATH=./enoterra_erp.db
VITE_API_URL=http://localhost:3001
```

### Продакшен (.env)
```env
NODE_ENV=production
PORT=3001
DB_PATH=./enoterra_erp.db
VITE_API_URL=https://yourdomain.com
```

## ⚠️ Важные правила

### ✅ Что можно делать
- Разрабатывать новые функции локально
- Тестировать на локальной БД
- Добавлять новые таблицы/колонки
- Изменять UI/UX
- Очищать локальную БД для тестирования

### ❌ Что нельзя делать
- Удалять существующие данные в продакшене
- Изменять структуру существующих таблиц без миграций
- Деплоить без тестирования
- Изменять продакшен БД напрямую
- Загружать локальную БД на сервер


## 📦 Скрипты для автоматизации

### 1. Очистка БД
```bash
node clean_database.mjs
```
- Удаляет все таблицы
- Создает чистую структуру
- Добавляет индексы
- Проверяет структуру

### 2. Проверка совместимости
```bash
node check_compatibility.mjs
```
- Сравнивает структуры БД
- Проверяет конфигурацию
- Создает отчет о совместимости

### 3. Безопасный деплой
```bash
node safe_deploy.mjs
```
- Создает резервные копии
- Проверяет совместимость миграций
- Собирает приложение
- Создает инструкции по деплою

### 4. Подготовка к деплою
```bash
node prepare_deployment.mjs
```
- Проверяет конфигурационные файлы
- Создает .env файлы
- Проверяет зависимости
- Создает документацию

## 🎯 Преимущества этой стратегии

1. **Безопасность данных**: Продакшен БД защищена
2. **Гибкость разработки**: Можно экспериментировать локально
3. **Простота деплоя**: Автоматизированный процесс
4. **Мониторинг**: Контроль за изменениями
5. **Откат**: Возможность вернуться к предыдущей версии
6. **Тестирование**: Полное тестирование перед деплоем

## 📞 Поддержка

При возникновении проблем:
1. Проверьте логи сервера
2. Сравните структуру БД локально и на сервере
3. Восстановите из резервной копии при необходимости
4. Проверьте совместимость версий
5. Используйте скрипт отката: `./rollback.sh`

## 🚀 Быстрый старт

### Для нового функционала:
1. `npm run dev` (в одном терминале)
2. `cd server && npm run dev` (в другом терминале)
3. Разрабатываете новые функции
4. `node check_compatibility.mjs` (проверка)
5. `node safe_deploy.mjs` (подготовка к деплою)
6. Загружаете файлы на Hostinger
7. Тестируете на продакшене

### Для исправления ошибок:
1. `node database_check.mjs` (проверка БД)
2. `node clean_database.mjs` (если нужно)
3. Исправляете ошибки
4. `node check_compatibility.mjs` (проверка)
5. Деплой на продакшен
